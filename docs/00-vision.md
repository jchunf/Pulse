# 00 — 产品愿景 Vision

> Pulse 是一个 macOS 本地优先（local-first）的后台常驻应用，把你"无意识"的电脑使用行为，可视化成一份属于你自己的**数字体检报告**。

## 一句话定位

**"数字体检 · 本地心跳 · 只有你自己看得到。"**

## 产品名解释

- **Pulse**（脉搏）— 电脑是你工作与生活的另一副身体，它每天都在产生数据，但很少有人定期"测一次脉搏"。Pulse 的使命就是让这些无形的数字行为变得可感知、可追溯、可对比。

## 核心价值主张

1. **被动采集，主动洞察** — 不需要手动打卡、不需要手动标记，打开电脑那一刻起，脉搏就在跳。
2. **本地优先，零上传** — 所有数据 100% 留在用户本地磁盘。Pulse 没有服务器账号、没有云同步、没有遥测。
3. **有趣 > 有用** — 数字工具已经足够多。Pulse 的差异化在于把"枯燥的统计"变成**有戏剧性的可视化**（里程表、彩蛋、鱼骨头、分身报告），让用户每天都愿意打开看一眼。
4. **隐私底线是第一特性，不是营销话术** — 键盘不记录内容、窗口标题默认模糊化、敏感时段可一键跳过。

## 目标用户画像

| 画像 | 特征 | 核心需求 |
|---|---|---|
| **技术创作者** | 程序员、设计师、写作者，每天在电脑前 6h+ | 想量化"今天到底产出了什么"、找出低效时段 |
| **自我量化爱好者** | 习惯用 Apple Health、Gyroscope、RescueTime 的人群 | 想要"数字版 Apple Health"：长期趋势 + 有趣彩蛋 |
| **隐私敏感用户** | 对云端打卡类工具（RescueTime、Toggl）有顾虑 | 必须本地化、可审计、可完全擦除 |
| **好奇心驱动用户** | 不一定有明确目标，但会被"你的鼠标已经跑了 47 公里"这种数据钩住 | 情绪化、娱乐化的数据展示 |

## 和同类产品的差异

| 产品 | 主要差异 |
|---|---|
| **RescueTime** | 云端、订阅制、只追踪应用时长。Pulse 本地 + 覆盖鼠标/键盘/系统状态的全维度。 |
| **Timing** | 专业强、可手动标注项目时间。Pulse 更"消费级"、重可视化彩蛋，不强调工时表单。 |
| **macOS 自带屏幕使用时间** | 只有应用分类 + 粗粒度时段。Pulse 深得多（鼠标/键盘/系统/环境）。 |
| **ActivityWatch（开源）** | 开源跨平台、数据足、但 UI 工程师友好而非用户友好。Pulse 重 UI/彩蛋/本地原生体验。 |

## 非目标（明确不做什么）

- **不做团队协作/工时汇报类功能** — 数据永远不出本地磁盘。
- **不做跨设备同步** — 不开服务端，不碰 iCloud。（导出/导入由用户自己完成。）
- **不采集屏幕内容/键盘字符/网络流量** — 这是底线，不是可配置项。
- **不上 macOS App Store** — 核心依赖（CGEventTap）在沙盒下被禁。
- **第一版不支持 Windows / Linux** — macOS 专注优先，其他平台长期路线再议。

## 成功定义（北极星指标）

- **D7 打开率** — 安装后 7 天内，用户平均每天主动打开 Pulse ≥ 1 次 → 证明可视化彩蛋确实让人上瘾
- **7 日留存** — 40%+ 用户在第 7 天仍在运行采集
- **"哇"时刻** — 用户至少有 1 次把 Pulse 的可视化截图分享到社交媒体（这是本地 app 唯一能被观察到的口碑信号）
