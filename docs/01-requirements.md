# 01 — 需求记录 Requirements

> 本文档记录 Pulse 在产品设计阶段从用户对话中提取的**原始需求**与**结构化条目**，所有后续功能与架构决策都应能追溯到这里的某一条需求。

## 一、需求来源

本产品从一次开放式对话中演进而来：
- 用户起始描述：希望有一个 macOS 后台常驻 app，采集"无意识的电脑使用行为"并做可视化报告
- 用户明确选择了一批功能作为首批范围
- 用户强调"有趣"是核心诉求之一
- 用户在多轮对话中反复强调**本地、不上传、隐私**

## 三、结构化需求（按维度归类）

### 3.1 功能需求 FR (Functional Requirements)

| ID | 需求 | 对应功能 ID |
|---|---|---|
| FR-01 | 追踪并可视化前台应用使用时长 | F-02 |
| FR-02 | 可视化一天/一周的活跃时段（热力图） | F-03 |
| FR-03 | 可视化鼠标活动轨迹（密度/散点） | F-04 |
| FR-04 | 累计鼠标移动距离并做有戏剧性的对比 | F-07 |
| FR-05 | 可视化键盘按键分布（QWERTY 热力图） | F-08 |
| FR-06 | 显示打字速度峰值 KPM | F-12 |
| FR-07 | 显示每日/每周趋势图（折线 + spark line） | F-01 |
| FR-08 | 实时仪表盘 | F-05 |
| FR-09 | 导出周报/月报 | F-06 |
| FR-10 | 连续记录激励（GitHub 风格贡献图） | F-11 |
| FR-11 | 每日时间带视图 | F-10 |
| FR-12 | 专注度指标 | F-09（待校准） |
| FR-13 | 一批扩展趣味功能（见 02-features.md B 组） | F-14 ~ F-33 |

### 3.2 非功能需求 NFR (Non-Functional Requirements)

| ID | 需求 | 验收标准 |
|---|---|---|
| NFR-01 | 本地优先，零数据上传 | 无任何出站网络调用（除了用户手动触发的更新检查） |
| NFR-02 | 后台持续运行，资源占用低 | 空闲时 CPU < 1%、内存 < 150MB |
| NFR-03 | 长期数据（年级别）不爆磁盘 | 重度用户 1 年 ≤ 100MB |
| NFR-04 | 键盘字符内容绝不记录 | 代码审计 + 红线检查清单 |
| NFR-05 | 窗口标题默认模糊化 | 默认只保留 App 名 + 标题哈希 |
| NFR-06 | 权限掉失能自检并提示 | 启动时校验 + 菜单栏视觉反馈 |
| NFR-07 | 数据可加密导出/导入 | 提供 CLI 或 UI 入口 |
| NFR-08 | 一键暂停采集（自动恢复） | 30 分钟超时自动恢复 |
| NFR-09 | **TDD** — 测试先行、红绿重构循环 | 新增功能 PR 必须附测试；核心模块覆盖率 ≥ 90% |
| NFR-10 | **CI** — 每次推送自动构建+测试+覆盖率报告 | PR 合并需 CI 绿灯；tag 触发自动签名/公证/发布 |
| NFR-11 | **用户中心** — 关键路径可用性优先于工程便利 | 首次启动 3 分钟内出现"有意义的第一眼"；主要流程 ≤ 3 次点击 |
| NFR-12 | **无障碍** — 支持 VoiceOver / Dynamic Type / 键盘导航 | 核心页面通过 macOS Accessibility Inspector 无警告 |

### 3.3 约束 C (Constraints)

| ID | 约束 | 来源 |
|---|---|---|
| C-01 | macOS 13+ 起支持（Ventura） | SMAppService API |
| C-02 | 不上 App Store | 沙盒禁 CGEventTap |
| C-03 | 必须 Developer ID 签名 + notarization | macOS 安全策略 |
| C-04 | Intel + Apple Silicon 双架构支持 | 目标用户覆盖 |
| C-05 | 不采集屏幕内容、网络流量、文件内容 | 隐私底线 |

## 四、需求演进与变更

| 日期 | 变更 | 原因 |
|---|---|---|
| 初版 | 键盘记录"按键计数 + 键码分布" | 用户 R-05 的初始表述 |
| 本版 | 默认只记录按键总数 + 时段聚合；键码分布改为可选 | 键码 + 时序有侧信道还原风险，见 `05-privacy.md` |
| 初版 | "专注度 = 应用切换频率" | 用户 R-09 |
| 本版 | 改为"按应用类别分组 + 用户可调阈值"（待用户确认） | 原定义会把看视频误判为"超级专注"，见 `09-open-questions.md` Q-01 |
| 初版 | Swift + Rust 守护进程 | 初稿技术栈猜想 |
| 本版 | 纯 Swift，不用 Rust | FFI 复杂度在 MVP 阶段不划算，见 `04-architecture.md` |
