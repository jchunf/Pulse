# 05 — 隐私设计 Privacy

> 隐私不是营销话术，是 Pulse 的**第一特性**。本文档定义隐私红线（绝不越过）、隐私分级、敏感数据处理细则、用户控制入口。实现阶段 PR 中涉及隐私相关改动**必须**引用本文档对应条款。

## 一、隐私第一原则（Principles）

1. **本地优先** — 数据默认不出用户磁盘。唯一合法的"出站"场景：用户主动点"检查更新"、用户主动点"导出报告"。
2. **最小化采集** — 能用聚合代替原始就用聚合；能用哈希代替明文就用哈希；能不采就不采。
3. **可暂停 / 可擦除 / 可导出 / 可关闭** — 四项用户控制必须全部提供。
4. **默认保守** — 默认开启的都是低敏感项；任何中/高敏感项默认关闭，用户显式开启。
5. **零遥测** — 不上报 crash 统计、不上报使用数据、不上报 IP。若用户主动上报 bug，由用户手动打包脱敏日志。
6. **可审计** — 源码应可公开，让隐私主张可验证（是否真正开源由商业策略决定，但代码结构要经得起审计）。

## 二、红线清单（永不采集）

以下内容 Pulse **永远不采集、不读取、不传输**，没有任何配置项可以开启：

| 类别 | 明细 |
|---|---|
| 键盘字符内容 | 任何明文字符、ASCII/Unicode 字符串 |
| 剪贴板内容 | 任何粘贴板上的文本、图片、文件引用 |
| 屏幕截图 | 不触发任何形式的屏幕捕获 |
| 文件内容 | 不读取用户文档的内容 |
| 网络流量 | 不监听网络请求、不记录 URL/域名 |
| 麦克风音频 | 不接入麦克风流（只观察"是否被其他 app 占用"） |
| 摄像头画面 | 同上 |
| 地理位置 GPS | 不调用定位 API；WiFi SSID 也不做地理反查 |
| 浏览历史 | 不读取任何浏览器的历史数据库 |
| iMessage / 邮件 | 绝不读取任何通讯应用的本地数据库 |

**实现保证**：代码审计清单（checklist）在 PR 模板里必选 —— 涉及以上类别的 import 需要双人 review。

## 三、隐私分级

| 等级 | 例子 | 处理策略 |
|---|---|---|
| **低** L | 应用使用时长、鼠标位移累计、按键总次数 | 可默认开启 |
| **中** M | 键码分布、窗口标题哈希、WiFi SSID 哈希、相机/麦克风占用状态 | 可默认开启但需提醒；部分默认关闭 |
| **高** H | 窗口标题明文、键码直方图的长期留存、日历事件标题 | **默认关闭**，需用户显式勾选 |

## 四、关键敏感项的处理细则

### 4.1 键盘

- **默认**：只存每秒按键次数（D-K1）、相邻按键间隔分布（D-K4）
- **可选进阶**：键码分布（D-K2） —— 用户在偏好里打勾才启用
- **为什么键码分布也要谨慎**：有研究表明 `keyCode + 时间戳` 的时序能部分还原文本内容（keystroke timing 侧信道），尤其在已知输入法的情况下
- **永远不存**：keyCode 对应的字符、修饰键组合的明文（仅在 D-K3 里做匿名化计数："cmd+c 按了 N 次"）

### 4.2 窗口标题

窗口标题是 Pulse 最敏感的数据之一 —— 它常常包含人名、邮件主题、文件名、聊天对象。

- **默认（D-A4）**：`SHA-256(title)` + `bundleId` + 首次出现时间戳；相同标题合并计数
- **明文（D-A5）**：默认关闭；用户在偏好里开启后才存明文
- **即使开了明文也支持"敏感关键词遮罩"**：用户配置的关键词（例如"离职"、"薪资"、"医院"）命中时，那条记录的标题改用掩码 `[REDACTED]`
- **推荐预设遮罩规则**：邮件主题前缀（"Re:", "Fwd:"）、某些特定 App（Messages、WhatsApp、Signal、1Password）**强制**仅存 App 名不存标题

### 4.3 剪贴板

- **只读 `NSPasteboard.changeCount` 的增量** —— 这个数字递增表示剪贴板被写入过
- **绝不调用** `readString`、`readData` 或任何读取 pasteboard 内容的 API
- **代码防护**：整个代码库禁止 import `NSPasteboard` 的内容读取路径，lint 规则强制

### 4.4 网络与位置（默认关闭）

- WiFi SSID（D-S6）：默认关闭。启用时需授权位置权限
- 即使启用，也只存 SSID 的 `SHA-256` 哈希（用户看到的是"你今天切换了 3 个地点"，不是地点名字）
- 地点名字本身由用户本地手动起别名（"家 / 公司 / 那家咖啡店"），别名存本地偏好

### 4.5 日历（默认关闭）

- 需 EventKit 授权
- 启用后，用户可在两种模式中选：
  - **仅时段模式**（推荐）：只记录"某段时间被 `is_meeting=true` 覆盖"，不存标题
  - **明文模式**：存事件标题（和标题遮罩规则一致）

### 4.6 通知统计

- 用系统 API 能拿到的是相对受限的计数（避免 private API 风险）
- 只记"被打断次数"，不记通知来源 App 或内容

## 五、用户控制入口

| 控制 | UI 位置 | 行为 |
|---|---|---|
| **一键暂停 30 min** | 菜单栏快捷项 + 全局快捷键（可自定义） | 暂停所有采集 30 分钟，自动恢复；期间菜单栏图标变色 |
| **敏感时段（自定义时长）** | 菜单栏 → "开始私密时段" → 选择 5/15/30/60 分钟 | 期间**完全不写库**（不是写后删） |
| **清除特定时段** | 偏好 → 数据 → "删除时间范围" | 删除选定范围内所有采集数据，跨所有表 |
| **清空所有数据** | 偏好 → 数据 → 危险区 → "清空并重置" | 需二次确认；保留偏好但清空 DB |
| **暂停单应用** | 右键菜单栏应用名 → "从此不记录 Xcode" | 该 bundleId 的任何数据都不再写入 |
| **关闭某采集项** | 偏好 → 采集 → 逐项开关 | 对应 D-XX 停止入库 |
| **加密导出** | 偏好 → 数据 → "备份到文件" | 弹密码 → 输出 `.pulse` 加密文件 |
| **完全卸载（含数据）** | 偏好 → 关于 → "卸载并清除所有本地数据" | 删 `~/Library/Application Support/Pulse/` 全部 |

## 六、隐私审计检查表（PR 必过）

涉及采集/存储/读取的 PR，作者必须在 PR description 里确认：

- [ ] 新增字段/表 已在 `03-data-collection.md` 注册
- [ ] 敏感分级（L/M/H）已标注
- [ ] 默认开/关已明确，中/高敏感默认必须关
- [ ] 是否触发权限申请；如是，已在 `06-onboarding-permissions.md` 同步
- [ ] 无 `NSPasteboard` 内容读取、无 `CGWindowListCreateImage`、无文件内容读取、无 URL 记录
- [ ] 用户控制入口是否覆盖（暂停/清除/关闭）
- [ ] 日志中不输出任何可能包含用户数据的字段（即使 debug log 也不行）

## 七、隐私承诺语（用户可见文案）

> **Pulse 承诺**
>
> - 你的数据 100% 留在你的 Mac 上。我们没有服务器。
> - 我们从不记录你打字的内容，也从不读取你的剪贴板。
> - 任何涉及窗口标题、键码、WiFi 的采集都是可关闭或默认关闭的。
> - 你可以在任何时候一键暂停、擦除或加密备份全部数据。
> - 我们不上报任何使用统计。检查更新也是你手动触发的。
>
> 如果哪天我们要改变以上任何一条，一定会先弹出显式提示并等你同意 —— 不会偷偷升级。

## 八、法律与合规占位

- **不向欧盟/加州等隐私法严格区域用户收集数据**：因为根本不收集云端数据，GDPR / CCPA 的"数据主体请求"对 Pulse 来说是空操作（可提供导出功能作为事实响应）
- **未成年人**：产品定位成人工作者，但不主动拒绝未成年人；文案保守
- **企业环境**：不主动支持 MDM 管控的合规审计（未来如做 for Team 版本再议）
