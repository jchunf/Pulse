# 11 — 用户中心设计原则 UX Principles

> 对应需求 R-21。本文档定义 Pulse 在 UI/交互/文案/无障碍方面的原则与硬指标，确保"技术上正确"不压过"用户真正愿意每天打开"。

## 一、核心原则

### P1 · 3 分钟内出现第一个"哇"

- 用户下载、授权、进入主界面到产生**第一个令其惊讶或愉悦的数据点**，不超过 3 分钟
- MVP 的实现：
  - 权限引导完成后立刻把"本次会话已经采集了 X 次鼠标点击、移动了 Y 毫米"显式放在欢迎页
  - 5 秒后更新一次数字，让用户看见它**在跳**
- 不等 24h 才出图表：首次启动时用"占位演示数据 + 明确标签"填满骨架，让用户看到"这里将来会展示什么"（非欺骗 —— 明确标注"示例数据，今日数据累积中"）

### P2 · 主要路径 ≤ 3 次点击

- 打开 app → 看到核心仪表盘：**1 次点击**（菜单栏图标）
- 查看任一指标详情：**≤ 2 次点击**（仪表盘 → 卡片）
- 调整偏好：**≤ 3 次点击**（菜单栏 → 偏好 → 目标项）
- 复杂功能（加密导出、清除时段）可以深（容错成本）；日常功能必须浅

### P3 · 视觉密度有节奏，不堆砌

- 仪表盘首屏**只放 4–6 个核心卡片**，每张卡片只有**一个关键数字 + 一张 spark line**
- 次级信息折叠在"展开详情"按钮后
- 拒绝"工业仪表盘"风格（一屏 20+ 指标），拒绝"营销首页"风格（无信息大图）

### P4 · 数据故事化，不是表格化

- "你今天用了 Xcode 4h 23min" → **不够**
- "你今天在 Xcode 里比昨天多写了 27 分钟，这是最近 7 天你最专注的编程日" → **可以**
- 每张卡片都要能回答"这个数字为什么值得看？它相对于什么变化？"
- 文案优先级：对比 > 绝对值 > 定义

### P5 · 绝不偷走注意力

- 不用红点徽章引诱点击
- 通知默认关闭（除了权限失效、严重错误）
- 不弹"打分/评价/分享"引诱
- 不在启动时弹"新版本更新说明"（放一个"什么是新的"小入口即可）
- 任何营销类文案必须能在**偏好里一键关闭**

### P6 · 数据不欺骗

- "今天使用 0 小时" 就显示 0 小时，不用"今天还没开始" 粉饰
- 不对齐小数位造假（`4.00h` 不写成 `4h`）
- 任何聚合口径变化（例如"活跃"定义修改）必须在变化那天显式标注
- 对比类的"北京→广州"这种有娱乐性，但必须附真实距离值（不能只写比喻）

### P7 · 允许用户"关掉"

- 任一可视化功能都能单独隐藏
- 任一采集项都能单独关闭
- 退出路径（暂停、删除数据、卸载）比入口更容易找到

### P8 · 空状态不空洞

- 首次安装 → 展示 3 天后"你会看到什么"的示意图
- 某个指标当日无数据 → "今天还没有键盘活动 —— 是在电话/会议中吗？" 而非 "暂无数据"
- 不用冷冰冰的"无数据"三个字

## 二、关键交互模式

### 菜单栏图标的"心跳"

- 正常运行时：图标有 2 秒一次的微动效（类似苹果原生 Siri 波纹）
- 暂停：心跳停止，图标变灰
- 权限失效：心跳停止 + 变红
- 敏感时段：心跳停止 + 变蓝（视觉不同于灰色，让用户知道是主动暂停而非故障）

### 首次启动的"第一眼"

```
┌─────────────────────────────────────────┐
│  你好。                                  │
│                                         │
│  过去 30 秒，你移动了鼠标 147 次，        │
│  累计 2.3 米 —— 相当于走了 3 步。        │
│                                         │
│  Pulse 正在记录你的数字脉搏。            │
│                                         │
│  [看看今天]  [先介绍一下]                 │
└─────────────────────────────────────────┘
```

这一屏的作用：**让用户立刻感受到"它在工作"**，而不是"它承诺将来会工作"。

### 里程表的"戏剧感"

不是：`鼠标移动距离：27,453 mm`

而是：
```
┌─────────────────────────────────────────┐
│         🏁                               │
│  你今天的鼠标跑了                        │
│                                         │
│        27.5 米                           │
│                                         │
│  ≈ 一个标准游泳池半长                    │
│  累计（近 30 天）：874 米                │
│  距离下一个里程碑「1 公里」：126 米       │
└─────────────────────────────────────────┘
```

## 三、本地化（L10n）

- MVP 默认支持：**英文 + 简体中文**（见 Q-15，用户拍板）
- 所有 UI 字符串必须走 `String(localized:)`，不允许硬编码
- 日期、时间、数字格式走 `Locale.current`
- 预留其他语言接入点（未来可能加繁中、日）

## 四、无障碍（a11y）

对应 NFR-12。Pulse 是个"给你看数据"的 app，必须考虑视觉障碍用户：

- **VoiceOver**：每个图表除可视化外，还要提供自然语言描述（`accessibilityLabel`）
  - 例如鼠标里程卡片："今日鼠标移动距离 27.5 米，相当于一个游泳池半长"
- **Dynamic Type**：字号跟随系统设置，不固定 pt
- **Keyboard Navigation**：所有交互元素可用 tab + 方向键访问
- **对比度**：文字对背景 ≥ WCAG AA（4.5:1）
- **动画尊重 reduceMotion**：关闭动画选项生效
- **测试**：每个 PR 必须在 Accessibility Inspector 无警告（CI 集成）

## 五、深色/浅色模式

- **默认跟随系统**
- 两套配色都要手动调过（不是 auto-generated）
- 数据可视化用的色板在两种模式下都有足够对比
- 用户可选"强制浅色/深色"（某些图表在深色模式下特定颜色不好看时提供逃生出口）

## 六、错误与回应

- **所有错误有人话翻译**：不直接展示 `Error domain: NSPOSIXErrorDomain code: 13`
- **错误 → 可操作**：每条错误必须给"下一步怎么办"（重试、查看日志、反馈）
- **首选原地修复**：能自动重试的不打扰用户
- **崩溃**：crash 恢复后首次打开告知"Pulse 上次异常退出，已恢复。期间 X 秒数据未采集。"
- **反馈入口**：在自检页底部放"报告问题"，预填系统信息（匿名化后）+ 一键打开邮箱

## 七、可用性测试计划

### MVP 冻结期（内测）

- 招募 **5–10 人**，目标用户画像匹配（见 `00-vision.md`）
- 录屏观察首次启动完整流程：安装 → 引导 → 首屏
- 记录前 7 天每人的打开频次
- 每人赠送 1 次 30min 远程访谈

### 关键问题清单

1. 引导流程中哪一步让你犹豫最久？
2. 首次看到仪表盘的第一反应是什么？
3. 7 天内最常看的是哪个卡片？最不会看的是哪个？
4. 有没有让你"哇"的瞬间？
5. 有没有让你删掉 app 的念头？为什么没删？

## 八、设计债务预警

以下情况要停下来而不是硬推 UI：

- 同一屏需要 ≥ 3 个横向滚动条
- 文案超过 3 行还解释不清楚
- 需要"用户教程"才能用 —— 除了首次 onboarding
- 必须在多个设置里横跳才能完成某件事
- "专业术语"出现在用户可见界面上（除非目标用户就是开发者，见 `00-vision.md`）

## 九、度量用户体验

除了北极星指标（见 `00-vision.md`），这些也关注：

| 指标 | 目标 | 来源 |
|---|---|---|
| 引导完成率 | ≥ 85% | 本地埋点（不上报，仅自检页可见） |
| 首次"哇"时间 | 中位数 < 3 min | 内测访谈 |
| 主界面每次停留 | ≥ 30s | 本地会话计时 |
| 每周触发偏好 | ≤ 1 次（说明默认足够好） | 本地埋点 |
| 卸载率 | < 30%（安装后 30 天） | 内测反馈 |
