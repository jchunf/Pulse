# 2026-04-17 — 产品方向评审

> 本文件不是设计规范，而是一次站在"MVP 基本完工（B1–B8 + A1–A15）"节点上的
> 产品视角评估，给接下来 v1.0 → v1.x 的优先级排序提供参考。结论可被后续
> 迭代推翻，但评估的基础观察应该保留备忘。

---

## 一、一句话评估

**底层采集和可视化已经扎实，离"让人持续用"还差一层关键的东西：从"数据"到"行为"的闭环。**

现在的 Dashboard 是被动的——用户得主动想起来打开它。对一个"后台常驻但没主动出声"的工具来说，这是留存的最大风险。

---

## 二、最该补的三件事（v0.1 → v1.0）

### 1. 留存钩子：让用户有理由每天回来看

**问题**：跑起来以后，菜单栏图标是静默的；Dashboard 除非手动打开，否则永远见不到。用户第一次看完"好酷啊我鼠标走了 87 米"之后，第二天想不起来还有这个 app。

**建议**：

- **每日脉搏简报**：清晨第一次解锁 Mac 时，菜单栏图标短暂动效 + 一个简短 summary window ——"昨天你敲了 12k 次键、鼠标走了 87 米、在 VS Code 里待了 5h"。类似 Screen Time 的周报，但每天都有、在你已经会看到的地方（菜单栏）。
- **周报仪式**：每周一在 `~/Library/Application Support/Pulse/reports/` 生成一份 HTML 报告，或者直接打开一个新 Window。用户点开那一瞬间就是价值兑现。
- **菜单栏异常徽章**：某指标比 7 日均值高 ±30%，图标加小红点。好奇心驱动回看。

**和现有功能关系**：`docs/02-features.md` 的 F-45（阈值告警）有类似规划，但"日报 / 周报"的**仪式感和被动 push** 比阈值告警更重要——后者容易被用户关掉通知，前者是一种期待。

### 2. 从"数字"到"故事"：F-25 Landmark 思路系统化

**观察**：现在**最有灵魂的一个功能是里程碑横幅**（F-25，"你的鼠标走了一个足球场"）——它把枯燥的数字变成了可分享的戏剧性瞬间。这个思路被严重低估了。

**建议**：把"戏剧化包装"抽象成一层，覆盖所有指标：

- **时间**："你今天用 Figma 的时间够看完《教父》"
- **按键**："你敲的字够写半部《活着》"
- **切换**："你今天在应用间跳了 247 次，平均每 2 分钟一次"（配一句 Pulse 式吐槽）
- **连续专注段**："你今天最长专注 73 分钟，发生在 14:12–15:25，当时你在 Xcode"
  - 这条尤其有价值。数据层从 B5 / B8 的 `min_switches` 应该已经能算出来，只是 UI 没展示。
  - 建议单开一张 summary 卡："Deep Focus Today"。

**为什么重要**：这是 Pulse 区别于 RescueTime / Toggl 的核心——**不是量化，是叙事**。冷数据谁都能算，但"你今天敲的字够写半部《活着》"是别人没有的。

实现上：把 LandmarkLibrary 的设计范式（金字塔距离 / 足球场）推广到所有维度，建一个统一的 `NarrativeEngine`，给任意数值 + 单位返回一句话包装。

### 3. 目标 / 意图层：让数据变成可行动的

**问题**：纯展示数据，用户感叹两句就关了。RescueTime 的留存差就输在这——它告诉你"你浪费了 3 小时"，然后呢？

**建议（极简，不要做成 habit tracker）**：

- Dashboard 顶部加一行："今天我想：[ ] 专注写代码 3h  [ ] 少切换应用  [ ] 不超过 1h 在 Slack"
- 晚上自动对比："目标 3h，实际 2h 41m ✅" / "❌ 差 19 分钟"
- **不强迫**：没设目标的用户什么都看不到；设了目标的用户多一个反馈回路
- **不做提醒打断**：达成与否只在 Dashboard 里显示，不推通知

**为什么重要**：把"看数据"变成"对镜子照自己"。用户从旁观者变成当事人，留存曲线会完全不同。

## 三、值得做但不急（v1.1+）

### 4. 上下文关联（Cross-metric insights）

单维度指标会腻。"在开会时段你的打字量下降了 60%"、"周二是你最低效的一天，连续 4 周如此"——这种 cross-metric 洞察**只有本地全量数据才能算**，是 Pulse 天生的护城河。

**重要警告**：不要用云端 LLM 总结。一旦出站调用任何 AI 服务，`05-privacy.md` 的全部承诺就崩了。

可选路径：
- 规则式 insight 引擎（if typing_rate < baseline * 0.6 && calendar.has_meeting）—— 简单、透明、可审计
- 等 MLX / llama.cpp on Metal 的 on-device 小模型稳定，再考虑 AI 摘要

两个路径都不急，规则式引擎先跑起来完全够用。

### 5. 导出 / hack 友好

目标用户是量化自我爱好者 + 开发者。他们会想**把数据接到别的地方**：

- `pulse export --from 2026-01-01 --format csv`
- 每日自动写一份 Markdown 到 Obsidian daily notes
- 一个 read-only JSON API（localhost only，bind 127.0.0.1）

这些**和本地优先原则完全兼容**，反而能形成社区黏性——有人会为了"能导出到 Obsidian"而装 Pulse。

### 6. 使用"姿势"分析

比"今天工作 8h"有意思得多的自我认知类指标：

- 键盘热力图（keycode 分布，已有 feature gate）——哪些键你用得最多
- Cmd+Tab vs 鼠标点 Dock 的比例 → "你是键盘党还是鼠标党"
- 平均每次 session 持续多久 → "你的工作节奏是 5 分钟一跳的 checker，还是 40 分钟一段的 deep worker"

### 7. 隐私"自证"工具

`05-privacy.md` 的 privacy pitch 是最大的差异化武器（vs RescueTime / Toggl），但要**证明**而不只是声称：

- **每次 release 附源码 tag**（项目开源已经做到了）
- **app 内"审计模式"**：Settings 加一个按钮 "Show what Pulse has recorded"，直接导出原始 sqlite 最近 1 小时的记录，让用户自己验证"你说不存打字内容，确实没存"
- **网络活动可视化**：app 内显示 Pulse 的出站连接——应该是 0（除了检查更新）。这种"自证"比文档里的承诺强 10 倍

## 四、建议警惕 / 先别做的

### 8. F-35 WiFi 画像 —— ROI 存疑

需要定位权限，弹窗会吓跑一部分用户。而"你在公司/家里工作的对比"这个 insight，用户自己一秒就知道，不需要 app 告诉他。

**建议**：除非有用户反复要求，v1.x 之前不做。

### 9. F-36 日历联动 —— 技术复杂度高，价值不确定

- 技术成本：循环事件、多日历账号（iCloud + Google + Exchange）、授权链路
- 价值前提：用户愿意授权、会议和行为确实强相关
- 真想做：先做一个最小版本，只读取"现在正在开会吗"这一个 bool，不碰事件标题内容

### 10. F-45 阈值告警 —— 做但要克制

"每小时弹窗告诉你玩了太久微信"会被秒关通知。做的话：

- 默认**关**
- 只在用户明确设置过阈值后才推
- 每天最多 1 条
- 永远不打断 macOS Focus / Do Not Disturb

## 五、优先级排序建议

以"用户真实下载到 Pulse 并留下来用"为目标，从现在（A15 完工）往下：

| 阶段 | 任务 | 理由 |
|---|---|---|
| **立刻** | 补完 `docs/06` 的 onboarding flow + 主动触发系统权限弹窗 | 用户流失的第一关口，现在还是被动提示 + 手动跳转 |
| **立刻** | 分发链路打通（Developer ID + notarization + Sparkle） | 没这步就没有真实用户，`docs/07` 已规划 |
| **v1.0** | 日报 / 周报推送机制（§2.1） | 解决留存，单一最重要的一块 |
| **v1.0** | Landmark 叙事扩展到所有指标 + "最长专注段"（§2.2） | 强化 Pulse 的差异化 |
| **v1.1** | 目标 / 意图层（§2.3） | 让数据可行动 |
| **v1.2** | 导出 / 审计工具（§3.5 + §3.7） | 让真爱粉沉淀 |

C 段的 WiFi / 日历 / 通知阈值，**先别碰**，等 v1.0 有真实用户反馈再决定。

## 六、底层观察备忘

这些观察在未来迭代中应该反复回来看，检查是否还成立。

1. **Pulse 的核心不是"准确的时间追踪"，是"有趣的自我画像"**。前者用户会拿它和 RescueTime 比精度，永远打不过；后者是别人没有的。
2. **"本地优先"是卖点，但只有用户感知得到才有价值**。需要主动展示（审计模式、网络活动可视化），不能只写在 docs 里。
3. **后台常驻 app 的宿命：要么被遗忘，要么被嫌烦**。走钢丝的艺术是"主动出现，但不打扰"——这正是日报 / 周报的设计核心。
4. **里程碑横幅（F-25）是目前最成功的设计**。它是 Pulse 式幽默和本地优先护城河的结合体，应该把这个思路最大化利用。
